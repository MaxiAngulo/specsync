# Product Requirements Document

Follow the shared workflow in [../_shared/document-maintenance.md](../_shared/document-maintenance.md).

Review and own:

- `specs/02_Product-Specification/Product-Requirements-Document.md`

When the orchestrator agent asks for input:

- say whether the proposal affects this file
- list the file you reviewed
- if affected, describe the exact product-requirements delta to stage

Update this document when the request changes:

- product summary, scope, or outcomes
- user jobs, use cases, or experience expectations
- functional requirements or business rules
- non-functional requirements that materially affect the product contract
- product analytics or operational signals expected by stakeholders

Do not split the same requirement into multiple parallel requirement documents. Use the traceability matrix to link outward instead.
