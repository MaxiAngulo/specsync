# Release Plan

Follow the shared workflow in [../_shared/document-maintenance.md](../_shared/document-maintenance.md).

Review and own:

- `specs/05_Operations-and-Delivery/Release-Plan.md`

When the orchestrator agent asks for input:

- say whether the proposal affects this file
- list the file you reviewed
- if affected, describe the exact release-plan delta to stage

Update this document when the request changes:

- release scope or target date assumptions
- deployment and validation steps
- rollout strategy, feature-flag plan, or communications
- go-live checklist items and release risks

Do not edit this document for implementation internals unless the release process or timing changes.
